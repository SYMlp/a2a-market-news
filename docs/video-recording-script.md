# 录屏脚本 — A2A 智选报社演示视频

> 目标时长：2-3 分钟
> 工具：OBS / 系统录屏 + 任意剪辑软件（剪映 / Premiere 等）
> 分辨率：1920x1080，浏览器全屏

## 录制前准备

### 1. 灌入演示数据

```bash
npx tsx prisma/seed-demo.ts
```

确认输出 6 个 apps + 10 条 feedbacks 都成功。

### 2. 环境检查

- [ ] `npm run dev` 启动本地服务
- [ ] 打开 `http://localhost:3000`，确认 Landing 页正常显示
- [ ] 用你自己的 SecondMe 账号登录一次，确认 PA 名字和头像正常
- [ ] 登出，回到 Landing 页准备开始录屏

### 3. 浏览器设置

- 隐藏书签栏、插件图标
- 关闭不相关的标签页
- 用 Chrome 全屏模式（F11）或窗口固定 1920x1080

## 段落 3：自动模式 — 大厅 → 日报栏

**时长**：~40 秒（实际录制约 2-3 分钟，后期加速）
**速度**：AI 等待时 4x 加速，对话出现时恢复 1x

| 步骤 | 操作 | 画面 |
|:---|:---|:---|
| 3.1 | 自动进入大厅 | 灵枢兔 🐰 开场白（NPC 气泡），PA 气泡显示名字 |
| 3.2 | PA 自主回应（等待 AI） | PA 思考动画（glow + THINKING）→ PA 气泡出现 |
| 3.3 | GM 路由 → 转场到日报栏 | 场景切换动画 |
| 3.4 | 编辑部助手 📰 推荐应用 | NPC 列出热门应用（来自 seed 的 6 个应用） |
| 3.5 | PA 自主选择一个应用体验 | PA 气泡："剧本杀局长听起来很有意思！" 之类 |
| 3.6 | 应用体验（外链跳转模拟） | 系统提示正在体验 → 返回 |

**后期字幕**：
- 3.2 时："PA 正在自主决策（没有人指导它）"
- 3.5 时："PA 根据自己的兴趣选择了应用"

**关键**：这段是全片核心。PA 的自主对话是最大亮点——观众要看到"PA 真的在自己做决定"。

## 后期剪辑要点

### 加速策略

| 场景 | 速度 | 理由 |
|:---|:---|:---|
| Landing 动画 | 1x | 动画本身就是亮点 |
| OAuth 登录 | 2-4x | 不是重点 |
| AI 思考等待（THINKING 状态） | 4x | 避免观众无聊 |
| 对话出现后 | 1x | 让观众读清内容 |
| 你在顾问模式打字 | 1x | 要展示人机交互差异 |
| 页面加载 / 路由跳转 | 2x | 非重点 |

### 字幕风格

- 位置：画面右下角或底部居中
- 字号：不超过画面宽度 40%
- 颜色：白底黑字半透明背景，或与场景 accent 色配合
- 停留时间：每条字幕至少 3 秒

### 背景音乐（可选）

轻快的 lo-fi 或 chiptune 背景音乐，音量控制在 10-15%，不盖过界面音效。

## 录制顺序建议

可以不按最终剪辑顺序录。建议先录最重要的段落（万一出错还有时间重录）：

1. **先录段落 3-4**（自动模式核心流程）— 最重要，可能需要多录几次
2. **再录段落 5**（顾问模式对比）— 第二重要
3. **再录段落 1-2**（开场 + 登录）— 几乎不会出错
4. **再录段落 6-7**（反馈 + 排行榜）— 稳定的静态页面
5. **最后录段落 8**（收尾告别）— 等前面都好了再录

### 多录备用素材

- 自动模式多录 2-3 次完整流程，选最好的一次
- 如果某次 PA 说了特别有个性的话，保留那段
- 如果不同 PA（不同 SecondMe 账号）能录一次对比，效果更佳
